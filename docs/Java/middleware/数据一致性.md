## 方案一（用的少 第一支持的少 然后很多的幂等性不能保证）业务流程 首先是业务方 发送一个半消息给mq mq返回发送成功给mq发送方 然后 业务方 执行本地事务  标识是成功或者是失败
 还有就是mq的server端要 能够回查事务的状态 因为可能有网络原因 这就是一个补偿机制 最后MQserver端是肯定知道 这个消息要不要投递给订阅方
